# Rogue Game

Консольное пошаговое roguelike-приложение на Java 21. Проект вдохновлен оригинальной игрой Rogue (1980) и реализует механику исследования процедурно генерируемых подземелий.

Для отрисовки псевдографического интерфейса в текстовом терминале (TUI) используется библиотека **Lanterna**.

---

## Запуск проекта

### 1. Сборка и запуск в Docker (Рекомендуемый способ)
Приложение полностью контейнеризировано. Сборка образа и запуск интерактивной TTY-сессии выполняются одной командой из корня проекта:

```bash
docker compose run --rm rogue-game
```
*Контейнер автоматически соберёт дистрибутив через Gradle, настроит окружение и запустит игру прямо внутри текущего терминала.*

### 2. Локальный запуск
Для сборки и запуска на хост-машине требуются установленный JDK 21 и настроенный Gradle.

```bash
# Сборка дистрибутива
./gradlew build

# Запуск приложения
./gradlew run
```

## Игровой процесс и управление

Игра работает в пошаговом режиме: мир реагирует и совершает действия только после хода игрока.

* **W / A / S / D** — Перемещение персонажа. Атака противника происходит автоматически при попытке сделать ход в занятую им клетку.
* **H** — Взаимодействие с оружием в инвентаре (выбор индекса 0–9).
* **J** — Использование еды для восстановления здоровья (выбор 1–9).
* **K** — Использование временных эликсиров (выбор 1–9).
* **E** — Чтение свитков для постоянного улучшения характеристик (выбор 1–9).

## План развития проекта (Roadmap)

В ближайших планах развитие и оптимизация следующих модулей:

1. **Независимость от раскладки клавиатуры:** Переработка слоя обработки ввода (`Presentation Layer`). Будет добавлена поддержка считывания скан-кодов клавиш или альтернативных символов (например, поддержка русских букв Ц/Ф/Ы/В вместо W/A/S/D), чтобы управление не ломалось при смене системной раскладки.
2. **Расширенная система сохранений (Save/Load):** Доработка полноценного интерфейса главного меню для ручной загрузки и сохранения текущего прогресса. Планируется реализовать поддержку нескольких слотов для сохранения, а также защиту от повреждения `JSON`-файлов при аварийном завершении игры.
